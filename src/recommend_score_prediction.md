# 추천 점수 예측 모델 정리

> 파일명: `recommend_score_prediction.ipynb`  
> 목적: `target_tourism_type.ipynb`에서 생성한 관광지 유형별 확률값과 외부 관광 수요 데이터를 활용하여 관광지별 추천 점수를 생성하고, 해당 점수를 예측하는 회귀 모델을 학습한다.

---

## 1. 모델 개요

본 노트북은 **관광지 추천 점수 예측 모델**을 구축하는 과정이다.  
이전 단계인 `target_tourism_type.ipynb`에서 생성한 관광지 유형별 확률값을 기반으로, 관광지의 기본 정보와 외부 관광 수요 지표를 결합하여 `recommend_score`를 생성한다.

현재 실제 사용자 평점, 클릭 로그, 예약 로그가 없기 때문에 `recommend_score`는 실제 정답 라벨이 아니라 공공데이터 기반의 **의사 라벨(pseudo-label)**이다.  
따라서 본 모델은 사용자의 실제 행동 데이터를 학습한 모델이라기보다, 관광지의 유형 특성, 정보 품질, 수요 지표를 조합해 추천 우선순위를 산정하는 모델에 가깝다.

본 노트북에서는 여러 회귀 모델을 동일한 조건에서 비교한 뒤, 평균 오차, 큰 오차에 대한 안정성, 전체 설명력을 종합적으로 고려하여 최종 추천 점수 예측 모델을 선정하였다.

---

## 2. 데이터 구성

### 2.1 기본 입력 데이터

| 구분 | 내용 |
|---|---|
| 기본 데이터 | `namyangju_dataset_with_type_probabilities.csv` |
| 생성 출처 | `target_tourism_type.ipynb`, 관광지 유형 추천 모델 |
| 주요 입력값 | 관광지 기본 정보, 유형별 확률, 대표 유형, 정보 품질 점수 |
| 주요 출력값 | `recommend_score`, `predicted_recommend_score`, `recommend_rank` |

`recommend_score_prediction.ipynb`는 `target_tourism_type.ipynb`의 결과 파일을 우선적으로 사용한다.  
특히 `type_prob_...` 형태의 관광지 유형별 확률 컬럼을 핵심 입력 변수로 사용한다.

---

### 2.2 외부 관광 수요 데이터

추천 점수를 생성하기 위해 다음과 같은 외부 데이터를 추가로 활용하였다.

| 데이터 | 활용 목적 |
|---|---|
| 방문자 수 추이 | 지역 관광 수요 규모 반영 |
| 방문자 증감률 | 최근 방문 수요 변화 반영 |
| 평균 체류시간 | 지역 체류 특성 반영 |
| 평균 숙박일 | 숙박 기반 관광 가능성 반영 |
| 숙박방문자 비율 | 체류형 관광 수요 반영 |
| 관광소비 추이 | 관광 소비 규모 반영 |
| 지역화폐 관광소비 추이 | 지역 내 소비 활성도 반영 |
| 내비게이션 목적지 검색량 | 관광지 유형별 관심도 반영 |

외부 데이터는 관광지 개별 단위 데이터라기보다 지역 또는 유형 단위의 요약 지표로 사용되었다.

## 3. 데이터 전처리

### 3.1 파일 탐색 및 CSV 로딩

노트북에서는 macOS 한글 파일명 문제와 CSV 인코딩 문제를 고려하여 다음과 같은 보조 함수를 사용한다.

| 함수 | 역할 |
|---|---|
| `normalize_text()` | 한글 파일명 정규화 |
| `read_csv_auto()` | `utf-8-sig`, `utf-8`, `cp949`, `euc-kr` 순서로 CSV 자동 로딩 |
| `find_file_by_keywords()` | 파일명 키워드를 이용해 필요한 CSV 자동 탐색 |

이를 통해 파일명 인코딩이나 한글 정규화 문제로 인한 로딩 오류를 줄였다.

---

### 3.2 관광지 유형 확률 데이터 불러오기

`target_tourism_type.ipynb`에서 생성된 관광지 유형별 확률 데이터를 불러온다.

| 컬럼 예시 | 의미 |
|---|---|
| `type_prob_관광지` | 관광지 유형일 확률 |
| `type_prob_자연관광` | 자연관광 유형일 확률 |
| `type_prob_역사관광` | 역사관광 유형일 확률 |
| `type_prob_문화관광` | 문화관광 유형일 확률 |
| `type_prob_음식` | 음식 유형일 확률 |
| `type_top1` | 가장 높은 확률의 대표 유형 |
| `type_top1_prob` | 대표 유형 확률 |
| `type_top2` | 두 번째로 높은 유형 |
| `type_top2_prob` | 두 번째 유형 확률 |

`type_top1`, `type_top2` 관련 컬럼이 없을 경우, `type_prob_...` 컬럼을 기준으로 자동 생성한다.

### 3.3 거리 관련 컬럼 제거

`recommend_score_prediction.ipynb`에서는 중심지 거리 기반 점수를 사용하지 않는다.  
따라서 다음과 같은 거리 또는 접근성 관련 컬럼은 제거하였다.

| 제거 컬럼 |
|---|
| `distance_from_namyangju_center_km` |
| `distance_from_center_km` |
| `access_score` |

거리 변수는 이후 `sequential_route_recommendation.ipynb`에서 **현재 위치 또는 숙소 기준 추천**을 계산할 때 별도로 반영한다.  
즉, `recommend_score_prediction.ipynb`는 관광지 자체의 기본 추천 점수를 만들고, `sequential_route_recommendation.ipynb`는 현재 위치와의 거리까지 고려한다.

### 4.2 정보 품질 점수

`info_score`는 관광지 정보의 완성도를 나타내는 지표이다.  
관광지명, 주소, 좌표, 이미지, 전화번호, 분류 코드 등의 존재 여부를 바탕으로 계산한다.

| 항목 | 반영 의미 |
|---|---|
| 관광지명 | 관광지 식별 가능성 |
| 주소 | 방문 가능성 |
| 좌표 | 지도 기반 추천 가능성 |
| 콘텐츠 유형 | 관광지 유형 구분 |
| 분류 코드 | 세부 유형 판단 |
| 이미지 | 웹 화면 표시 품질 |
| 전화번호 | 문의 및 예약 연계 가능성 |
| 수정일 | 데이터 최신성 판단 |

`info_score`는 실제 관광 매력도를 직접 의미하지 않으므로 추천 점수 계산 시 낮은 비중으로 반영하였다.

`modifiedtime`을 날짜형으로 변환한 뒤, 최근에 수정된 데이터일수록 높은 값을 갖도록 `recent_score`를 생성하였다.

| 파생 변수 | 의미 |
|---|---|
| `recent_score` | 관광지 정보가 최근에 수정되었는지를 나타내는 점수 |

이 값은 관광지 정보 관리 상태를 간접적으로 반영한다.

## 5. 외부 관광 수요 요약 변수 생성

외부 관광 통계 데이터는 개별 관광지와 직접 연결하기 어려운 경우가 많기 때문에, 지역 또는 유형 단위의 평균값으로 요약하여 모델 입력에 추가하였다.

| 변수 | 의미 |
|---|---|
| `avg_monthly_visitors` | 평균 월별 방문자 수 |
| `avg_visitor_growth_rate` | 평균 방문자 수 증감률 |
| `avg_stay_time` | 평균 체류시간 |
| `avg_overnight_days` | 평균 숙박일수 |
| `avg_overnight_ratio` | 평균 숙박방문자 비율 |

해당 변수들은 관광지 개별 특성보다는 남양주시 관광 수요의 전반적인 수준을 반영한다.

## 6. 내비게이션 검색량 매핑

내비게이션 목적지 유형별 검색량을 관광지 유형과 매핑하여 `nav_score`를 생성하였다.

### 6.1 처리 방식

| 단계 | 내용 |
|---|---|
| 1 | 내비게이션 목적지 검색량 데이터 로딩 |
| 2 | 목적지 유형별 평균 검색량 계산 |
| 3 | 검색량을 0~1 범위로 정규화 |
| 4 | 관광지의 콘텐츠 유형명, 대표 유형, 타겟 유형과 매칭 |
| 5 | 관광지별 `nav_score` 생성 |

`nav_score`는 관광지 또는 관광지 유형에 대한 실제 검색 관심도를 반영하는 지표로 사용된다.

## 7. 추천 점수 생성

### 7.1 추천 점수 구성 요소

`recommend_score`는 다음 요소를 가중합하여 생성하였다.

| 요소 | 설명 | 가중치 |
|---|---|---:|
| `nav_score` | 내비게이션 목적지 검색 관심도 | 0.30 |
| `type_quality_score` | 관광·역사·문화·체험·자연·레저 계열 유형 적합도 | 0.20 |
| `type_top1_prob` | 대표 유형 예측 확률 | 0.15 |
| `info_score` | 데이터 정보 완성도 | 0.10 |
| `recent_score` | 데이터 최신성 | 0.10 |
| `regional_demand_score` | 지역 관광 수요 요약 점수 | 0.15 |

### 7.2 추천 점수 공식

```text
recommend_score =
100 × (
  nav_score × 0.30
  + type_quality_score × 0.20
  + type_top1_prob × 0.15
  + info_score × 0.10
  + recent_score × 0.10
  + regional_demand_score × 0.15
)
```

이때 중심지와의 거리 기반 점수는 제외하였다.  
거리 요인은 사용자의 숙소 또는 현재 위치에 따라 달라지므로 모델 3에서 별도로 계산한다.

### 8.2 타겟 변수

| 구분 | 변수 |
|---|---|
| 타겟 변수 | `recommend_score` |

`recommend_score`는 실제 사용자 평가 점수가 아니라 공공데이터와 관광지 속성을 기반으로 생성한 의사 라벨이다.

### 8.3 전처리 방식

| 피처 유형 | 전처리 |
|---|---|

---

### 8.4 비교 모델

추천 점수 예측은 연속형 점수를 예측하는 회귀 문제이므로 여러 회귀 모델을 비교하였다.

| 모델 | 특징 |
|---|---|
| RandomForestRegressor | 기존 기준 모델, 비선형 관계에 안정적 |
| ExtraTreesRegressor | RandomForest보다 더 강한 랜덤성을 적용한 앙상블 모델 |
| GradientBoostingRegressor | 순차적으로 오차를 줄여가는 기본 부스팅 회귀 모델 |
| HistGradientBoostingRegressor | 대용량 데이터에 효율적인 히스토그램 기반 부스팅 모델 |
| CatBoostRegressor | 범주형 데이터와 비선형 패턴에 강한 부스팅 모델 |
| XGBoostRegressor | 예측 성능이 높은 대표적인 그래디언트 부스팅 모델 |

모든 모델은 동일한 입력 피처, 동일한 학습/테스트 데이터 분할, 동일한 전처리 파이프라인을 기준으로 비교하였다.

---

### 8.5 최종 적용 모델

모델 비교 결과, `XGBoostRegressor`는 MAE 기준에서 가장 낮은 평균 절대 오차를 기록하였다.  
그러나 `GradientBoostingRegressor`는 RMSE가 가장 낮고 R²가 가장 높게 나타나 큰 오차에 대한 안정성과 전체 설명력 측면에서 가장 우수하였다.

두 모델의 MAE 차이는 약 0.006으로 매우 작기 때문에 평균 오차 측면에서는 큰 차이가 없다고 판단하였다.  
따라서 본 모델에서는 평균 오차, 큰 오차에 대한 안정성, 전체 설명력을 종합적으로 고려하여 `GradientBoostingRegressor`를 최종 추천 점수 예측 모델로 선정하였다.

```python
GradientBoostingRegressor(
    n_estimators=300,
    learning_rate=0.05,
    max_depth=3,
    random_state=42
)
```

| 파라미터 | 의미 |
|---|---|
| `n_estimators=300` | 300개의 부스팅 트리 생성 |
| `learning_rate=0.05` | 부스팅 단계별 학습률 |
| `max_depth=3` | 각 트리의 최대 깊이 제한 |
| `random_state=42` | 실험 재현성 확보 |

## 9. 모델 평가 결과

학습 데이터와 테스트 데이터는 8:2 비율로 분할하였다.  
추천 점수 예측 모델은 연속형 점수인 `recommend_score`를 예측하는 회귀 문제이므로, 모델 평가는 MAE, RMSE, R²를 기준으로 수행하였다.

| 지표 | 의미 | 판단 기준 |
|---|---|---|
| MAE | 실제 점수와 예측 점수의 평균 절대 오차 | 낮을수록 좋음 |
| RMSE | 큰 오차에 더 민감한 평균 제곱근 오차 | 낮을수록 좋음 |
| R² | 추천 점수 변동을 모델이 설명하는 정도 | 높을수록 좋음 |

---

### 9.1 모델별 성능 비교

| 순위 | 모델 | MAE | RMSE | R² |
|---:|---|---:|---:|---:|
| 1 | GradientBoosting | 0.1921 | 0.3823 | 0.9980 |
| 2 | XGBoost | 0.1861 | 0.4200 | 0.9975 |
| 3 | ExtraTrees | 0.2405 | 0.5701 | 0.9954 |
| 4 | HistGradientBoosting | 0.2758 | 0.5195 | 0.9962 |
| 5 | RandomForest | 0.2855 | 0.5952 | 0.9950 |
| 6 | CatBoost | 0.3119 | 0.6961 | 0.9932 |

XGBoost는 MAE가 0.1861로 가장 낮아 평균 절대 오차 기준에서는 가장 우수한 성능을 보였다.  
반면 GradientBoosting은 RMSE가 0.3823으로 가장 낮고, R²가 0.9980으로 가장 높게 나타났다.

MAE 기준에서 XGBoost와 GradientBoosting의 차이는 약 0.006으로 매우 작다.  
그러나 RMSE와 R² 기준에서는 GradientBoosting이 더 우수하므로, 큰 오차를 줄이고 전체 추천 점수 변동을 설명하는 측면에서 GradientBoosting이 더 안정적인 모델로 판단된다.

따라서 본 연구에서는 평균 오차뿐만 아니라 큰 오차에 대한 안정성과 전체 설명력을 함께 고려하여 `GradientBoostingRegressor`를 최종 추천 점수 예측 모델로 선정하였다.

---

### 9.2 최종 선택 모델 성능

| 지표 | 값 |
|---|---:|
| 최종 모델 | GradientBoostingRegressor |
| MAE | 0.1921 |
| RMSE | 0.3823 |
| R² | 0.9980 |

최종 선택된 GradientBoostingRegressor는 평균 절대 오차 0.1921을 기록하였다.  
추천 점수가 0~100점 범위로 구성되어 있다면, 이는 평균적으로 약 0.19점 정도의 오차로 추천 점수를 예측한다는 의미이다.

RMSE는 0.3823으로 전체 비교 모델 중 가장 낮게 나타났으며, 이는 큰 오차 발생 가능성이 상대적으로 낮다는 것을 의미한다.  
R²는 0.9980으로 가장 높게 나타났으며, 이는 입력 변수들이 추천 점수의 변동을 대부분 설명하고 있음을 의미한다.

다만 `recommend_score` 자체가 가중합 기반으로 생성된 의사 라벨이므로, 높은 R²를 실제 사용자 만족도 예측 성능으로 해석해서는 안 된다.  
현재 성능은 **공공데이터 기반 추천 점수 산식의 재현 성능**으로 보는 것이 적절하다.

## 16. 최종 정리

본 노트북은 `target_tourism_type.ipynb`에서 생성한 관광지 유형별 확률값을 활용하여 관광지별 추천 점수를 생성하고, 이를 예측하는 회귀 모델을 구축하였다.  
추천 점수는 내비게이션 목적지 검색 관심도, 관광지 유형 적합도, 대표 유형 예측 확률, 정보 완성도, 데이터 최신성, 지역 관광 수요 지표를 가중합하여 생성하였다.

모델 학습에는 RandomForest, ExtraTrees, GradientBoosting, HistGradientBoosting, CatBoost, XGBoost를 비교 대상으로 설정하였다.  
모든 모델은 동일한 입력 피처, 동일한 학습/테스트 데이터 분할, 동일한 전처리 파이프라인을 기준으로 비교하였다.

| 모델 | MAE | RMSE | R² |
|---|---:|---:|---:|
| XGBoost | 0.1861 | 0.4200 | 0.9975 |
| GradientBoosting | 0.1921 | 0.3823 | 0.9980 |
| ExtraTrees | 0.2405 | 0.5701 | 0.9954 |
| HistGradientBoosting | 0.2758 | 0.5195 | 0.9962 |
| RandomForest | 0.2855 | 0.5952 | 0.9950 |
| CatBoost | 0.3119 | 0.6961 | 0.9932 |

XGBoost는 MAE 기준에서 가장 낮은 평균 절대 오차를 기록하였다.  
그러나 GradientBoosting은 RMSE와 R² 기준에서 가장 우수한 성능을 보였으며, XGBoost와의 MAE 차이도 약 0.006으로 매우 작았다.

본 모델의 목적은 관광지별 추천 점수를 안정적으로 예측하고 이를 기반으로 추천 순위를 산정하는 것이다.  
따라서 평균 오차뿐만 아니라 큰 오차에 대한 안정성과 전체 설명력을 종합적으로 고려하여 `GradientBoostingRegressor`를 최종 추천 점수 예측 모델로 선정하였다.

다만 `recommend_score`는 실제 사용자 만족도나 평점이 아니라 공공데이터 기반으로 생성한 의사 라벨이다.  
따라서 모델 성능은 실제 관광객 만족도 예측 성능이라기보다, 정의한 추천 점수 산식을 얼마나 잘 재현했는지를 의미한다.

최종 결과물인 `predicted_recommend_score`와 `recommend_rank`는 모델 3의 숙소/현재 위치 기반 순차 관광지 추천 엔진에서 기본 추천 점수로 활용된다.
